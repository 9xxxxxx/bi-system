# BI 报表与通报编辑系统需求规格

## 1. 目标与范围

本系统面向团组内部的数据分析与经营通报工作，首期打通“文件导入、数据治理、数据集建模、仪表盘制作、通报编写、发布与导出”的完整闭环。系统应满足中小规模团队当前使用，并为后续推广到更多团组或部门保留组织、权限、认证和部署扩展能力。

首期界面使用简体中文；代码、接口和数据库字段使用英文。文案必须通过国际化资源管理，为后续增加语言包预留能力。

## 2. 用户与权限

系统提供可组合的职责角色：

- 系统管理员：管理成员、角色、工作区、系统配置和审计。
- 数据管理员：管理文件导入、数据集、指标、质量规则和数据批次。
- 内容编辑者：创建仪表盘、模板和通报草稿。
- 发布者：发布、撤回通报并创建正式修订版。
- 查看者：查看和导出已授权内容。

资源可设为仅自己、指定成员或角色、整个工作区、指定权限组可见。数据集、仪表盘和通报分别控制查看、编辑、共享、导出和发布权限；所有查询、AI 分析、导出和通报快照必须叠加行级数据权限。首期采用单组织、单工作区运行，但核心资源需保留组织和工作区归属。

## 3. 核心业务流程

1. 数据管理员上传 Excel 或 CSV，选择固定模板或通用导入向导。
2. 系统预览工作表与字段，执行映射、类型识别和数据质量校验。
3. 用户选择追加、按业务主键更新或全量替换；系统保留原始文件、导入批次、错误行和处理结果。
4. 数据管理员通过可视化关联构建数据集，维护公共指标和计算字段。
5. 内容编辑者从空白画布或模板创建仪表盘，配置图表、筛选、联动和下钻。
6. 内容编辑者在结构化通报中引用实时指标或图表，编辑期间可刷新数据。
7. 具备发布权限的用户发布通报；系统冻结数据批次、指标版本、筛选条件和图表配置。
8. 成员在系统内查看，或按权限导出正式文件和图片。

## 4. 数据接入与质量

固定模板导入用于稳定、周期性数据；通用导入向导支持选择工作表、字段映射、类型转换、空值处理和导入模式。单文件上限为 100 MB 或约 100 万行，导入需分块处理并展示进度，不得将整个文件一次性载入内存。

质量规则包括必填、唯一、类型、长度、范围、枚举、正则、跨字段约束、业务主键重复及批次波动检查。规则分为错误和警告：错误阻止入库，警告经授权确认后可继续。系统生成质量报告与错误行文件；存在严重质量问题的数据不得用于发布通报。

首期不直接连接外部数据库，也不重新开发 API 取数。现有 API 查询脚本作为后续候选集成，届时通过适配接口接入，固定 Cookie 由管理员手工更新并作为加密凭据管理。

## 5. 数据集与指标

数据集支持事实表与维度表的可视化关联、内连接、左连接、关联字段和一对一/一对多基数声明，并提示重复聚合风险。已保存的关联模型可被多个数据集复用。

正式经营指标进入统一指标库，记录名称、定义、公式、单位、适用维度、负责人、版本和变更历史。数据集或图表可创建临时计算字段，验证后可提升为公共指标。首期支持求和、平均、计数、去重计数、最大值、最小值、四则运算、同比、环比、累计、排名、占比、目标完成率和时间进度；不允许执行任意代码。

## 6. 仪表盘与分析交互

系统支持空白画布、预设模板和团队模板。模板实例化后可自由修改，管理员可将仪表盘发布为新模板。桌面端与移动端分别保存布局；移动端支持查看、筛选、下钻和导出，不提供编辑。

首期组件覆盖指标、比较、趋势、占比、分布、流程、地理、数据和内容类别，包括 KPI、趋势指标、目标进度、柱状图、条形图、堆叠图、瀑布图、折线图、面积图、组合图、饼图、环图、矩形树图、旭日图、散点图、气泡图、箱线图、直方图、热力图、漏斗图、桑基图、仪表盘、地图、明细表、交叉表、排行表、富文本和图片。

编辑器支持拖拽字段、聚合、排序、Top N、双轴、堆叠、条件格式、阈值、目标线、参考区间、自定义单位、图例、标签、提示信息、主题、多页面导航以及组件复制粘贴。筛选支持全局、页面和组件范围，包含级联、联动、相对日期、参数控件、面包屑、个人视图和书签。

分析交互必须支持层级下钻与上钻、携带筛选上下文的明细穿透，以及点击图表过滤其他组件。所有交互均继承权限和数据批次上下文。

## 7. 智能分析

首期支持规则告警、趋势预测、异常点检测、贡献度分析、自动指标摘要、自然语言问数、自动生成图表和对话式分析。AI 生成的图表或文字只能由用户主动保存，不能自动发布。

首期仅使用外部云模型。数据字段分为公开、内部、敏感和严格敏感四级；发送前必须执行掩码、泛化、去标识化、聚合和小样本保护。模型不得直接连接数据库，查询必须在系统内完成权限、指标、只读、复杂度和结果规模校验。普通数据校验后可自动执行，敏感数据执行前需用户确认，严格敏感数据禁止 AI 问数。结果必须展示数据集、指标口径、筛选条件、生成时间和失败原因，不得在证据不足时编造答案。

## 8. 通报编辑与发布

通报采用结构化块编辑，支持标题、段落、列表、表格、图片、指标、图表、目录、分隔和分页。模板支持封面、期号、日期、部门等变量，固定、可选和重复章节，必填与锁定内容块，页眉、页脚、页码、水印、落款和统一样式。模板升级不得修改既有通报。

首期使用单人编辑锁，提供自动保存、版本历史、超时释放、管理员接管和冲突恢复。草稿中的指标与图表可刷新；发布时生成不可变快照。首期不设置审核流程，编辑者保存草稿，具备独立发布权限的成员执行发布或撤回。修改已发布通报时必须创建新版本。

## 9. 导出、通知与审计

仪表盘整体支持 PNG、PDF 导出，单图支持 PNG 与复制图片；通报支持 PDF、Word、PNG 长图和打印；表格及明细支持 Excel、CSV。导出任务在后台运行，并记录数据批次、筛选条件、生成者和生成时间。

系统内通知覆盖内容共享和发布、导入结果、质量异常、指标告警、编辑锁变化和导出完成。首期不接入邮件、短信或企业消息平台。

审计记录覆盖认证、账号、权限、数据、指标、仪表盘、通报、导出、AI、配置、备份和迁移操作，默认保留 180 天。记录不可由普通管理员修改，不得包含 Cookie、令牌、密码或未脱敏业务数据，并支持检索与 CSV 导出。

## 10. 非功能要求

- 规模：中小规模数据，约 20 名用户同时在线。
- 性能：普通页面 2 秒内可交互；缓存仪表盘 3 秒内展示主要组件；常规查询 5 秒内返回；复杂查询超过 10 秒显示进度并受超时限制；常规联动 2 秒内反馈。验收以标准数据集下的 P95 为准。
- 可移植性：当前在 Windows 开发运行，后续迁移 Linux 服务器时不得修改业务代码；禁止依赖盘符、桌面路径或操作系统特有行为。
- 安全：不开放注册，账号由管理员创建；密码安全哈希、首次改密、失败锁定、会话过期、强制下线和敏感操作确认。首期不要求双因素认证。
- 浏览器：支持当前主流稳定版 Chrome 与 Edge，移动查看支持现代 Android 与 iOS 浏览器。
- 可恢复性：原始文件按内容去重；核心配置采用压缩、低峰、限速备份，默认保留 3 个每日和 2 个每周备份。分析表允许由原始文件重建，危险迁移前执行完整备份。
- 可观测性：关键请求、后台任务、导入、查询、导出和 AI 调用具有结构化日志、状态、耗时和可定位错误编号。

## 11. 错误与状态处理

导入任务至少包含待处理、处理中、成功、部分成功、失败和已取消状态；失败重试不得造成重复数据。后台导出与智能分析采用相同的可查询任务状态。用户可取消尚未进入不可逆阶段的任务。

通报状态为草稿、已发布和已撤回；已发布版本不可覆盖。删除的数据集、仪表盘和通报先进入回收站，默认保留 30 天。资源被引用时，删除操作必须展示影响范围并阻止产生悬空引用。

面向用户的错误必须说明发生位置、影响和可执行的恢复动作；敏感内部信息只写入受控日志。

## 12. V1 交付分段

已确认的首期能力共同构成 V1，但必须按以下里程碑递进交付，每个里程碑单独形成实现计划、测试报告和可运行版本：

- M0 工程基础：版本管理、前后端工程、环境配置、数据库迁移、质量工具、自动化测试和部署骨架。
- M1 数据接入：固定模板、通用导入、批次、质量规则、错误报告和可恢复导入。
- M2 数据建模：多表关联、数据集、公共指标、计算字段、行级权限和查询服务。
- M3 BI 核心：仪表盘编辑、模板、核心与标准图表、筛选、桌面和移动布局。
- M4 高级分析：联动、下钻、穿透、参数、书签、高级图表配置、告警和性能优化。
- M5 通报系统：结构化编辑、模板、编辑锁、快照发布、版本、导出和通知。
- M6 智能与加固：自然语言问数、预测、异常分析、脱敏网关、审计完善及跨平台验收。

后一个里程碑可以依赖前一个里程碑的稳定接口，但不得以未完成的未来模块作为当前验收前提。本次工程化工作只实施 M0；其余里程碑在 M0 完成后逐项设计和实施。

## 13. V1 验收标准

1. 使用真实样本完成固定模板和通用向导导入，并验证三种导入模式、错误报告和可追溯批次。
2. 从多表关联模型创建公共指标和临时指标，计算结果与人工核算一致。
3. 从模板和空白画布分别创建桌面、移动仪表盘，完成筛选、联动、下钻、穿透、书签和权限验证。
4. 将图表与指标插入通报，完成自动保存、发布快照、修订、撤回及 PDF、Word、PNG 导出。
5. 不同角色和行级权限用户查询、AI 问数、导出同一资源时只能获得授权数据。
6. 使用标准数据集和并发场景达到第 10 节性能目标，100 MB 文件导入过程中服务保持可用。
7. 在 Windows 完成开发运行验证，并通过可移植部署流程在 Linux 兼容环境完成启动、迁移、备份和恢复演练。

## 14. 明确不在 V1 范围

首期不包含直接数据库连接、现有 API 脚本集成、审核工作流、实时多人协作、外部通知、本地大模型、自定义图表插件、移动端编辑、原生移动应用、透视分组、多租户计费和独立 OLAP 引擎。上述能力必须通过后续需求评审进入版本计划。
